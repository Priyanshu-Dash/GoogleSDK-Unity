## 🔍 Verify EDM4U is Working

### Step 1: Check EDM4U Menu
In Unity:
1. Go to **Assets** → **External Dependency Manager** → **iOS Resolver**
2. You should see options like:
   - **Settings**
   - **Resolve**
   - **Install Cocoapods**

### Step 2: Check iOS Resolver Settings
1. **Assets** → **External Dependency Manager** → **iOS Resolver** → **Settings**
2. Verify these settings:
   - ✅ **Podfile Generation** - Should be enabled
   - ✅ **CocoaPods Integration** - Should be enabled
   - ✅ **Pod Tool Path** - Should point to your CocoaPods installation

### Step 3: Verify Dependencies.xml Location
The file should be at:
```
Assets/GoogleSignIn/Plugins/iOS/Dependencies.xml
```

It should contain:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<dependencies>
  <iosPods>
    <iosPod name="GoogleSignIn" version="~> 7.0" />
  </iosPods>
</dependencies>
```

## 🚀 Build iOS Project

### What Happens Automatically

When you build the iOS project from Unity:

1. **EDM4U Detects Dependencies.xml**
   - Reads `Assets/GoogleSignIn/Plugins/iOS/Dependencies.xml`
   - Identifies Google Sign-In SDK requirement

2. **EDM4U Generates Podfile**
   - Creates Podfile in Xcode project directory
   - Includes Google Sign-In SDK dependency

3. **EDM4U Runs pod install**
   - Automatically executes `pod install`
   - Downloads and installs Google Sign-In SDK
   - Installs all dependencies (AppAuth, GTMAppAuth, etc.)

4. **EDM4U Creates Workspace**
   - Generates `Unity-iPhone.xcworkspace`
   - Links Unity project with CocoaPods dependencies

5. **PostProcessBuild Configures Info.plist**
   - Adds URL scheme from GoogleService-Info.plist
   - Configures OAuth callback

### Build Steps

1. **In Unity:**
   - **File** → **Build Settings**
   - Select **iOS**
   - Click **Build** (or **Build and Run**)
   - Choose export location

2. **Watch Unity Console:**
   - You should see EDM4U messages:
     - "iOS Resolver: Generating Podfile..."
     - "iOS Resolver: Running pod install..."
     - "iOS Resolver: Pod installation complete"

3. **After Build:**
   - Navigate to exported Xcode project
   - You should see:
     - ✅ `Podfile` (generated by EDM4U)
     - ✅ `Pods/` directory (with dependencies)
     - ✅ `Unity-iPhone.xcworkspace` (ready to open!)

## 📱 Open and Build in Xcode

1. **Open Workspace:**
   - Double-click `Unity-iPhone.xcworkspace`
   - **NOT** `Unity-iPhone.xcodeproj`

2. **Build in Xcode:**
   - Select your target device/simulator
   - Click **Play** button
   - Should build successfully!

## ⚠️ Important Notes

### GoogleService-Info.plist Required

Before the app will work, you need:

2. Place it in `Assets/` folder (same level as `google-services.json`)

## 🔧 Troubleshooting

### EDM4U Not Running pod install

**Solution:**
1. **Assets** → **External Dependency Manager** → **iOS Resolver** → **Resolve**
   - This manually triggers dependency resolution

2. Check settings:
   - **Assets** → **External Dependency Manager** → **iOS Resolver** → **Settings**
   - Ensure "Podfile Generation" is enabled

### Still Getting Header Not Found Error

**Solution:**
1. Make sure you opened `.xcworkspace` (not `.xcodeproj`)
2. Clean build folder in Xcode: **Product** → **Clean Build Folder**
3. Rebuild

### Dependencies Not Installing

**Solution:**
1. Check CocoaPods is installed: `pod --version` in Terminal
2. If not: `sudo gem install cocoapods`
3. Then: **Assets** → **External Dependency Manager** → **iOS Resolver** → **Install Cocoapods**
